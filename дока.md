1.  ОБЩИЕ ПРАВИЛА
    ------------------------------------------------------------
    - Все элементы, компоненты, шрифты и цвета берутся **только из этого стайлгайда**.
      Если каких-то элементов, цветов или шрифтов не хватает — напиши мне, я добавлю.
    - Экранная ширина: **360 пикселей** (мобильный viewport).
    - Рабочая область контента: **328 пикселей** (отступы от краёв экрана по 16px).
    - Высота экрана может меняться в зависимости от количества контента.
      Минимальная высота экрана: **640 пикселей**.

        --color-divider: #DCE0E6 (обводки, разделители)

3.  COMPONENTS & THEIR USAGE
    --------------------------
    BUTTON
        - .btn (Primary): Главное действие на экране (оплатить, продолжить, сохранить).
          Состояния: default, pressed (фон #11A34F), disabled.
        - .btn-secondary (Secondary): Альтернативное действие (отмена, назад, нет).
          Состояния: default, pressed (фон #EDF0F4), disabled.
        - .btn-short (Short): Пара коротких кнопок для модальных окон (Да/Нет).
          Отступ между кнопками: 16px.
        - .btn-product (Product Actions): Действия по продукту (Перевести/Пополнить).
          Размер: 160×56px, иконка на зелёном круге. Отступ между парой: 8px.
        - .sbp-banner (SBP Banner): Информационная плашка, некликабельная.
          Используется для отображения платёжного метода СБП.

    INPUT
        - .floating-input (Text): Поле ввода текста с плавающим лейблом. Высота 56px.
          Используется для логинов, паролей, сообщений.
        - .floating-input-sum (Sum): Поле для ввода суммы. Высота 64px, шрифт 20px.
          Используется в переводах и платежах.
        - .preview-field-wrapper (Preview): Не редактируемое поле (лейбл + текст + линия).
          Используется для отображения данных без возможности изменения.

    SELECTION
        - .checkbox-label (Checkbox): Квадрат 20×20px, скругление 6px.
          Используется для множественного выбора (выберите несколько вариантов).
        - .radio-label (Radio): Круг 20×20px, скругление 50%.
          Используется для выбора одного варианта из нескольких.
        - .switch-label (Switch): Трек 36×20px, тумблер 16px.
          Используется для включения/выключения опции.

    TAB BAR
        - .tab-bar: Контейнер для табов. Отступ между текстами: 30px.
        - .tab-item: Неактивный таб, цвет text-gray.
        - .tab-item.active: Активный таб, цвет primary, подчёркивание 2px.
          Используется для переключения между разделами на одной странице.

    PRODUCT SELECT
        - .product-select-wrapper: Контейнер.
        - .product-select: Плашка 328×64px, border-radius: 6px, padding: 9px 16px.
          Используется для выбора счёта или карты перед платежом или переводом.
          Состояния: default, .focus (обводка primary), .error (обводка error),
                     .disabled (фон disabled).

    DROPDOWNS (выпадающие списки)
        - .dropdown-wrapper: Контейнер.
        - .dropdown-field: Поле выбора (64px, аналогично Product Select).
        - .dropdown-content: Выпадающий список. Высота 200px, скролл 4×32px.
        - .dropdown-item-product: Плашка выбора продукта (308×52px, иконка, название, сумма).
        - .dropdown-item-icon-text: Плашка выбора банка (308×56px, иконка, текст).
        - .dropdown-item-text: Плашка выбора периода (308×36px, только текст).
          Используются для выбора из предложенного списка.

    MODALS
        - .modal (Modal): Обычное модальное окно. Ширина 328px, обводка 1px divider,
          тень 0 4px 16px rgba(0,0,0,0.08). Используется для подтверждения действий
          (удалить, выйти, отправить).
        - .driver-sheet (Driver): Нижний лист. Ширина 360px, скругления сверху 16px,
          обводка 1px divider. Используется для навигации по продуктам или выбора
          из нескольких вариантов.

4.  UI/UX & USABILITY GUIDELINES FOR MOBILE BANKING
    ------------------------------------------------------------
    A.  МОБИЛЬНЫЕ UI/UX ПРИНЦИПЫ
        - **Размеры касания (Touch targets)** — все интерактивные элементы должны быть
          ≥ 48×48px (кнопки, поля, иконки) для удобства пальцев.
        - **Иерархия контента** — наиболее важная информация и действие — вверху экрана,
          второстепенное — ниже. Главный экран: баланс, быстрые действия, последние операции.
        - **Консистентность** — одинаковые элементы выглядят и работают одинаково во всех
          экранах (используй токены стайлгайда).
        - **Обратная связь** — любое действие пользователя должно сопровождаться визуальной
          реакцией: нажатие кнопки (pressed), загрузка (spinner), сообщения об успехе/ошибке.
        - **Минимизация шагов** — для частых действий (платёж, перевод) — не более 3 шагов.

    B.  ЮЗАБИЛИТИ (Usability)
        - **Предсказуемость** — одинаковые элементы ведут себя одинаково.
        - **Доступность** — контраст текста ≥ 4.5:1, размер шрифта ≥ 12px, поддержка
          масштабирования шрифта (увеличь шрифт через настройки устройства).
        - **Помощь и подсказки** — подсказывай формат ввода (например, «__ __ __» для карты),
          показывай примеры, используй плейсхолдеры.
        - **Предотвращение ошибок** — валидация полей в реальном времени (окраска обводки
          в error), блокировка кнопки «Отправить» при незаполненных обязательных полях.
        - **Гибкость и эффективность** — предоставляй возможность сохранять шаблоны для
          частых платежей, использовать автозаполнение.

    C.  БАНКОВСКАЯ СПЕЦИФИКА (Banking-specific)
        - **Безопасность** — критичные действия (подтверждение платежа, изменение лимитов)
          требуют дополнительного подтверждения (PIN, биометрия, SMS-код).
        - **Доверие** — всегда отображай состояние счёта/карты (баланс, лимиты).
          Скрывай конфиденциальные данные (полные номера карт) по умолчанию.
        - **Финансовая информация** — суммы и валюты должны быть чёткими, крупными.
          Используй цвет `primary` для рублей, `disabled` для копеек и валюты.
        - **Ошибки** — сообщения об ошибках должны быть конкретными и предлагать решение:
          «Неверный пароль. Попробуйте ещё раз или восстановите доступ» вместо просто «Ошибка».
        - **Платежи и переводы** — всегда показывай сумму списания и итоговый баланс после
          операции. Предлагай сохранить шаблон для частых платежей.
        - **История операций** — должна фильтроваться по дате, типу, сумме, с возможностью
          поиска. Каждая операция должна содержать: дату, время, сумму, тип, описание.

    D.  НАВИГАЦИЯ И СТРУКТУРА
        - **Нижняя навигация (Tab Bar)** — используй для основных разделов (Главная, Платежи,
          История, Профиль). Активный таб — подчёркивание 2px, цвет primary.
        - **Верхняя навигация** — заголовок экрана, кнопка «Назад» (если нужно), кнопки действий
          (редактировать, добавить, настройки).
        - **Модальные окна** — для подтверждения действий (удалить, выйти, отправить).
        - **Драйверы (Action Sheets)** — для выбора из нескольких вариантов (выбор продукта,
          выбор банка, выбор периода).

    E.  ЭКРАНЫ (Screen Layouts)
        - **Главный экран** — сверху: приветствие и баланс. Средняя часть: быстрые действия
          (кнопки Перевести, Пополнить, Платежи). Нижняя часть: последние операции.
        - **Экран платежа** — выбор продукта (Product Select), ввод суммы (Input Sum),
          кнопка Продолжить (Primary), подсказка о комиссии (если есть).
        - **Экран истории** — фильтры (период, тип, сумма), список операций (каждая строка:
          дата, время, сумма, тип, описание). Возможность поиска.
        - **Экран профиля** — аватар, имя, телефон, настройки (уведомления, лимиты, безопасность).

5.  GENERAL RULES FOR AI
    ----------------------
    - Всегда используй CSS-переменные для цветов и типографики.
    - Для кнопок Primary используй класс .btn, для Secondary — .btn-secondary.
    - Поля ввода (Input) всегда используй с плавающим лейблом (.floating-label).
    - Для выбора продукта используй .product-select (он же используется в дропдаунах).
    - В дропдаунах для полей выбора используй .dropdown-field вместо .product-select, чтобы избежать конфликтов.
    - Модальное окно (.modal) и драйвер (.driver-sheet) должны иметь обводку 1px solid var(--color-divider).
    - Для состояний (focus, error, disabled) добавляй соответствующие классы.
    - Всегда проверяй, что вёрстка соответствует мобильному viewport (360px / 328px).
    - Следуй принципам юзабилити и банковской специфике, описанным в разделе 4.
